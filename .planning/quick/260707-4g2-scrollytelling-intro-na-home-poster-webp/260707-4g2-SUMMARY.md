**Scroll-scrubbed all-intra car video (poster -> video crossfade) with a white->dark veil that reveals the existing home search UI, coupled to app.js only via a body[data-state] MutationObserver.**

## Accomplishments
- Re-encoded the 10s/240-frame source clip to an all-intra 1920x1080 H.264 file
  (240/240 keyframes, 27.6 MB) enabling O(1) `video.currentTime` seeks for smooth
  scroll-scrubbing.
- Added the intro DOM contract (`intro-scroller` / `intro-stage` / poster / video /
  veil / hint) and CSS section 12 without touching or duplicating the existing
  `.home-view` markup.
- Implemented a self-contained `intro.js` controller: rAF-lerped video scrubbing,
  buffered-based poster->video crossfade, veil + home-view reveal, mouse parallax,
  and a `MutationObserver`-driven finish path that requires zero changes to `app.js`.

## Files Created/Modified
- `static/media/carro-scroll.mp4` - all-intra re-encode (240/240 keyframes, 27.6 MB, 1920x1080, no audio) for scrub-friendly seeking
- `static/media/carro-poster.webp` - instant first-frame poster (frame 0 == video frame 0, invisible crossfade seam)
- `templates/index.html` - head script sets `dataset.intro` from `prefers-reduced-motion`; `<html data-intro="active">` default; intro scroller/stage/poster/video/veil/hint markup before `.home-view`; `intro.js` loaded before `app.js`
- `static/css/style.css` - new "12. Scrollytelling intro" section: safe-default hidden `.intro-scroller`, active-state 500vh scroller, sticky white `.intro-stage`, poster/video crossfade, veil, hint with CSS arrow, and the active-state `.home-view` centered-overlay reveal rules
- `static/js/intro.js` - IIFE controller: scroll-progress -> video-time mapping (disassemble 0->0.5, reassemble 0.5->1), rAF lerp loop, buffered-based crossfade with canplay fallback, veil/reveal/parallax driven from progress, idempotent `finishIntro()`, `MutationObserver` on `body[data-state]`

## Decisions Made
- Kept the plan's ffmpeg command as-is; the 720p downscale fallback branch was not
  needed since the all-intra 1080p output (27.6 MB) came in well under the ~42 MB cap.
- Added a static `data-intro="active"` attribute directly on `<html>` (on top of the
  head script's `dataset.intro` assignment) so the attribute is present in the raw
  markup, not only set at runtime — belt-and-suspenders against any no-JS or
  pre-hydration flash, and keeps the DOM contract self-documenting.

## Deviations from Plan

None - plan executed exactly as written. The only addition (static `data-intro="active"`
default on `<html>`) is consistent with the plan's own stated goal of "prevents any flash
of the wrong layout" and does not change behavior (the head script overwrites it
synchronously before first paint).
