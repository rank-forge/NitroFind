<task type="auto">
  <name>Task 3: Implement the scroll-scrub controller in static/js/intro.js</name>
  <files>static/js/intro.js</files>
  <action>
    Create a self-contained IIFE (vanilla, no libs). It runs at end-of-body (DOM already
    parsed). Bind to the Task-2 DOM contract by id/class.

    Guard + refs: read `document.documentElement.dataset.intro`; if it is not `"active"`
    (reduced-motion or already done), return immediately (home stays static). Cache
    `.intro-scroller`, `.intro-stage`, `#intro-poster`, `#intro-video`, `.intro-veil`,
    `.intro-hint`, `.home-view`. If any required element is missing, call the finish path
    and return.

    Scroll progress: `progress = clamp(window.scrollY / (scroller.offsetHeight - window.innerHeight), 0, 1)`.

    Video time mapping (disassemble then reassemble):
    - Read `duration` from the video's `loadedmetadata` event; fall back to 10 (seconds).
    - For progress p: if p ≤ 0.5 → targetTime = (p / 0.5) * duration; else → targetTime =
      (1 - (p - 0.5) / 0.5) * duration. So p:0→0.5 plays 0→duration (car opens), p:0.5→1
      plays duration→0 (car reassembles).

    rAF loop (single loop, started once): NEVER set currentTime directly from the scroll
    event. Keep a `renderedTime` variable and lerp it toward `targetTime` by ~0.14 each
    frame. Only assign `video.currentTime = renderedTime` when `video.readyState >= 2`,
    the video is not currently `.seeking`, and the delta from the last assigned value is
    above a small epsilon (avoid seek thrashing). Continuously drive:
    - Veil opacity: ramp 0→1 as progress goes 0.82→0.95 (clamped).
    - Home-view reveal: opacity 0→1 as progress goes 0.86→1.0; transform
      `translateY(Npx)` from ~24px→0 over the same range; set `pointerEvents = 'auto'`
      once opacity crosses ~0.9 (so the search input becomes clickable at the end).
    - Parallax: lerp current offset toward the mouse target and apply
      `transform: translate3d(x, y, 0)` (~±8px) plus an optional tiny rotate to
      `.intro-stage`. Skip parallax math when reduced-motion (already guarded out).
    Stop the rAF loop when finished.

    Poster→video crossfade: listen for `progress`/`canplaythrough` on the video; when the
    buffered ranges cover (roughly) the full duration — helper checks
    `video.buffered.end(last) >= duration - 0.3` — add the `.ready` class to the video so
    CSS crossfades it over the poster (frame 0 == poster, so it is invisible). Also add a
    fallback timeout so a `canplay` that never reports full-buffer still crossfades.

    Mouse parallax input: a `mousemove` listener stores a normalized target offset from the
    viewport center; the rAF loop consumes it (do not transform inside the event).

    Scroll hint: fade `.intro-hint` opacity to 0 once `window.scrollY > 40`.

    Completion / finish path (`finishIntro()`): idempotent. Triggered when progress ≥ ~0.995.
    It: sets `document.documentElement.dataset.intro = 'done'`; clears the inline
    opacity/transform/pointerEvents it set on `.home-view` (so `.home-view` returns to its
    native static centered CSS); cancels the rAF loop; removes scroll/mouse listeners;
    and `window.scrollTo(0, 0)` (the page collapses to ~100vh once the scroller is hidden,
    so the search sits centered with no leftover scroll).

    Robustness:
    - Video failure: on the video `error` event, do NOT abort the intro — leave the poster
      visible and keep the veil + reveal + scroll working over the static poster.
    - Leaving home: attach a `MutationObserver` to `document.body` watching the `data-state`
      attribute; if it becomes anything other than `"home"`, call `finishIntro()`. This makes
      a first search/Escape/back cleanly consume the intro so returning to home shows the
      search centered with NO forced re-scroll — and requires ZERO changes to app.js.
    - Never call `video.play()`; the video is muted/playsinline and only seeked.

    Keep all math in small pure helpers (clamp, lerp, mapProgressToTime, isFullyBuffered).
  </action>
