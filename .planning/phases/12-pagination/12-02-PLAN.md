<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser client → Flask `/api/search` | Untrusted `page` query-string value crosses here and becomes an ES `from_` offset |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01 | Tampering | `page` param → ES `from_` | mitigate | `page = max(1, _safe_int_param(request.args.get("page")) or 1)` — non-integer strings coerce to `None`→1; ES never receives a raw user string; `from_` is a computed int |
| T-12-02 | Denial of Service | out-of-range `page` (e.g. `page=0`, `page=999999`) | mitigate | `page=0` clamps to 1 (`from_=0`, non-negative). Very large `page` yields `from_ > max_result_window`; ES returns a 400 which the existing `try/except` converts to HTTP 500 — no crash, no data leak, no amplification |
| T-12-03 | Information Disclosure | `resp["hits"]["total"]["value"]` in wrapper response | accept | `total` is an integer hit count only — no document content or PII; exposing the count is the intended PAGE-02 behavior |
| T-12-SC | Tampering | package installs | accept | No packages installed; single-file edit to `nitrofind/server.py` — no supply-chain surface |
</threat_model>
