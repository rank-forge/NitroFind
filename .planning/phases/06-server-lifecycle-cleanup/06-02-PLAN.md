<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → Flask route | Each request reads the in-memory `state["ready"]` flag set by the background thread |
| Flask process → ES subprocess | `_es_health_poller` spawns and polls the ES JVM on localhost:9200 |
| background thread → state dict | Single writer (poller) / many readers (route handlers, Flask threaded mode) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05 | Information Disclosure | `/api/status` before ES ready | mitigate | 503 warmup guard checks `state["ready"]` before any ES client call — never queries cat.indices while starting (RESEARCH.md anti-pattern) |
| T-06-06 | Tampering | shared `state` dict (concurrent reads) | accept | Single writer, simple bool/int/str values; GIL makes read/write atomic — no Lock needed (RESEARCH.md) |
| T-06-07 | Denial of Service | ES poll loop never terminates | mitigate | Hard 180s `time.monotonic()` deadline (D-03); daemon thread cannot block process exit |
</threat_model>
