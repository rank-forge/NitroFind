<objective>
Add the sort control UI: three toggle buttons in the results filter row that let the user reorder results by relevance (default), newest (date), or largest (size). The buttons send the `sort=` param the backend (Plan 01) already accepts.

Purpose: SORT-01 — give users direct control over result ordering through the browser UI, mirroring the existing persistent-filter pattern.

Output: Sort button markup in `templates/index.html`, `currentSort` state + `onSortChange` handler + sort param wiring in `static/js/app.js`, and `.sort-btn` styling in `static/css/style.css`. A human-verify checkpoint confirms the buttons render, toggle active state, and reorder live results (per 10-VALIDATION.md manual-only checks).
</objective>

## Phase Goal

Users get correct results despite typos and can control result ordering (frontend layer: sort toggle buttons wired to the `sort=` API param).
