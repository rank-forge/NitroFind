<objective>
Add frozen-mode plumbing to nitrofind/es_manager.py and main.py so the
application can launch from a PyInstaller bundle on a clean machine: compute
ES_HOME from sys.executable, inject elasticsearch.yml + jvm.options into the
bundled ES directory at startup, and harden the ES subprocess against the
windowed-mode `[WinError 6]` failure mode. Ship unit tests that pin the new
contracts before any frozen build is attempted.

Purpose: This is the runtime side of PKG-01. The frozen build script and
distribution archive (Plan 02) only work if the launcher correctly resolves
the bundled ES path, writes the correct config, and spawns the ES subprocess
without inheriting the windowed app's null std handles.
Output: Three new functions / behaviors in nitrofind/es_manager.py, two
targeted edits to main.py, and a tests/test_packaging/ subtree with three
test modules covering resolve_es_home, inject_es_config, and the subprocess
kwargs contract.
</objective>

New symbols this plan adds to nitrofind/es_manager.py:
- resolve_es_home() -> str | None
    Pure function. In frozen mode (getattr(sys, 'frozen', False)) returns
    str(Path(sys.executable).parent.glob('elasticsearch-8.*')[0]) or None.
    In dev mode returns os.environ.get('ES_HOME').
- inject_es_config(es_home: str, config_src_dir: str) -> None
    Idempotent. Copies {config_src_dir}/elasticsearch.yml -> {es_home}/config/elasticsearch.yml.
    Copies {config_src_dir}/jvm.options    -> {es_home}/config/jvm.options.d/nitrofind.options,
    creating jvm.options.d/ via os.makedirs(..., exist_ok=True) first.
