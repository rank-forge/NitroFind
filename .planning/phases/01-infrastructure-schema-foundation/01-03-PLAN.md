SpinnerWidget contract (UI-SPEC.md Qt-Specific Implementation Notes, Pattern 5):

    - Fixed size: 48 x 48 px (setFixedSize(48, 48))
    - Transparent background: setAttribute(Qt.WidgetAttribute.WA_TranslucentBackground)
    - QTimer with 100ms interval drives rotation
    - _angle += 30 per tick (modulo 360); self.update() triggers repaint
    - paintEvent: QPainter with antialiasing on; QPen color #26a69a, width 4px, RoundCap; drawArc with rect = self.rect().adjusted(4, 4, -4, -4); start angle = _angle * 16, sweep = 120 * 16 (Qt expects 1/16th degree units)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input (Retry / Quit clicks) -> LoadingWindow signals | Click events from the local user; not an adversarial boundary on a single-user desktop tool |
| pyqtSignal cross-thread delivery (Plan 04 wires ESHealthWorker -> show_error) | Qt serializes signal arguments across thread boundaries automatically; reason strings are app-controlled, not user-supplied |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | LoadingWindow Quit button | accept | Quit button calls `QApplication.instance().quit` directly. A malicious local user could click Quit to exit, but this is the intended behavior — no privilege boundary crossed. |
| T-03-02 | Tampering | LoadingWindow.show_error(reason) | mitigate | The `reason` string is rendered into a QLabel via `setText` — Qt's QLabel does NOT execute HTML/JS by default. Even if Plan 04 passes through unsanitized JVM error text, no script injection is possible. If rich text rendering is ever enabled (`setTextFormat(Qt.TextFormat.RichText)`), this mitigation must be revisited. |
| T-03-03 | Information Disclosure | Error message label | mitigate | Error messages displayed are the two static strings from the Copywriting Contract (mapped from worker reasons in Plan 04). Raw JVM stack traces and exit codes are NOT shown to the user (UI-SPEC Copywriting rules: "Do not include exit codes, stack traces, or Java JVM output in the visible label — log those to stderr only"). |
| T-03-04 | Denial of Service | SpinnerWidget QTimer | accept | Timer ticks 100ms; if the widget is leaked the timer continues. Parented to self at construction so destruction of the window automatically stops the timer. Risk accepted given the bounded number of LoadingWindow instances (exactly one per app lifecycle). |
</threat_model>
