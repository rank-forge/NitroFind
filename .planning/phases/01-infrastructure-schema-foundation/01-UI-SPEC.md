# Phase 1 — UI Design Contract

> Visual and interaction contract for the Phase 1 loading window.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope note:** Phase 1 UI is MINIMAL — one window only: the loading/splash window shown
> while Elasticsearch warms up. The main search UI (search bar, results, filters, detail pane)
> is out of scope here and will be fully designed in Phase 4. Do not extend this contract
> into Phase 4 territory.

## Spacing Scale

Qt QSS does not use CSS custom properties. Spacing tokens below map directly to `setContentsMargins`,
`setSpacing`, and fixed `setFixedSize` calls in Python code. All values are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between label and spinner |
| sm | 8px | Vertical spacing between spinner and status text |
| md | 16px | Padding inside the loading window content area |
| lg | 24px | Vertical spacing between the app title and the spinner group |
| xl | 32px | Window-level internal margin (top and bottom) |
| 2xl | 48px | Minimum window side padding |
| 3xl | 64px | Not used in Phase 1 (reserved for Phase 4 layout) |

Exceptions: Loading window fixed size is 360 × 240px (not derived from spacing scale — fixed to
provide a stable, centered splash appearance). Source: default (no upstream specification; 360×240
is the Qt convention for splash/loading dialogs on desktops with ≥1080p resolution).

## Typography

Qt QSS font declarations use `font-size` (pt or px), `font-weight`, and `line-height` is set via
`QLabel` height or `setMinimumHeight`. All sizes below are in **points (pt)** to respect system
DPI scaling — Qt renders pt sizes correctly on HiDPI displays; px bypasses DPI scaling.

| Role | Size | Weight | Line Height | QSS Property |
|------|------|--------|-------------|--------------|
| App title ("NitroFind") | 20pt | 600 (SemiBold) | 1.2 (tight) | `font-size: 20pt; font-weight: 600;` |
| Status text ("Starting search engine...") | 11pt | 400 (Regular) | 1.5 | `font-size: 11pt; font-weight: 400;` |
| Error message | 11pt | 400 (Regular) | 1.5 | Same as status text |
| Button labels (Retry, Quit) | 10pt | 600 (SemiBold) | n/a (button height fixed) | `font-size: 10pt; font-weight: 600;` |

Typography uses exactly **2 sizes in active UI** (20pt title, 11pt body) + 1 button size (10pt).
Weights: Regular (400) and SemiBold (600) only.

> Source: default (standard Qt desktop conventions); CONTEXT.md D-06 (branding + status text confirmed);
> REQUIREMENTS.md UIPL-03 (dark theme confirmed)

## Component Inventory

Phase 1 requires the following Qt components. This is the complete inventory — no additional
components are in scope.

| Component | Qt Class | State(s) | Notes |
|-----------|----------|----------|-------|
| Loading window | `QWidget` (frameless, fixed size 360×240) | loading, error | Not `QSplashScreen` per D-06 |
| App title label | `QLabel` | static | Text: "NitroFind" at 20pt SemiBold |
| Animated spinner | Custom `QWidget` subclass using `QTimer` + `QPainter` | spinning, hidden | 48×48px; arc animation, 100ms tick, 30° step per tick |
| Status text label | `QLabel` | loading_text, error_text | Transitions between the two states |
| Retry button | `QPushButton` | visible only in error state | Hidden during loading state |
| Quit button | `QPushButton` | visible only in error state | Hidden during loading state |
| ES health worker | `QThread` subclass | polling, ready, failed | Emits `es_ready` or `es_failed(reason: str)` signals |

Layout: `QVBoxLayout` centered in the window — title → spinner → status text → button row
(hidden until error). Button row uses `QHBoxLayout` with equal stretch on both sides for centering.

## State Machine

The loading window has exactly 2 states:

### State 1: Loading (initial state)

- Spinner: visible, animated (rotating arc)
- Status text: "Starting search engine..."
- Retry button: hidden (`setVisible(False)`)
- Quit button: hidden (`setVisible(False)`)

**Transition to State 2:** `QThread` emits `es_failed(reason)` (timeout at 60 s or non-zero exit code)

### State 2: Error

- Spinner: hidden (`setVisible(False)`)
- Status text: error message (see Copywriting Contract below)
- Retry button: visible, accent-colored background
- Quit button: visible, destructive-colored label

**Transition back to State 1:** User clicks Retry. Handler calls `process.terminate()`, resets
polling, restarts `QThread`, transitions status text back to "Starting search engine...".

**Exit:** User clicks Quit → `QApplication.quit()`

**Success (no State 3 visible):** `QThread` emits `es_ready` → loading window closes → main window
(Phase 4, not in scope here) is shown. In Phase 1, the main window placeholder can be a minimal
`QMainWindow` with title "NitroFind — Ready" as a stub.

> Source: CONTEXT.md D-06 (loading state), D-07 (error state), D-03 (QThread architecture),
> D-04 (60s timeout), D-05 (shutdown logic)

## Copywriting Contract

| Element | Copy |
|---------|------|
| App title | NitroFind |
| Loading status text | Starting search engine... |
| Error message — timeout | Could not connect to Elasticsearch. Check that ES_HOME is set correctly and try again. |
| Error message — crash | Elasticsearch exited unexpectedly. Check your ES_HOME directory and try again. |
| Retry button label | Retry |
| Quit button label | Quit |
| Destructive confirmation | None — Quit requires no confirmation dialog (it is a startup-failure exit, not a data-destructive action) |

Error message rules:
- Maximum 2 sentences.
- First sentence states the problem. Second sentence states the corrective action.
- Do not include exit codes, stack traces, or Java JVM output in the visible label — log those to stderr only.

## Qt-Specific Implementation Notes

These notes are for the executor only — they are part of the design contract for this Qt phase.

### Applying the Theme

```python
from qt_material import apply_stylesheet
app = QApplication(sys.argv)
apply_stylesheet(app, theme="dark_teal.xml")
```

Apply BEFORE creating any window. This injects global QSS that all widgets inherit.

### Spinner Implementation

Use `QTimer` + `QPainter` arc. Do NOT use `QProgressBar` in indeterminate mode — its animation
is visually inconsistent across platforms. A custom `QWidget` gives pixel-accurate control.

Spinner spec:
- Widget size: 48×48px fixed (`setFixedSize(48, 48)`)
- Arc pen width: 4px
- Arc color: accent `#26a69a`
- Arc span: 120° (one-third of circle)
- Rotation speed: 30° per timer tick, 100ms interval (360° / 1200ms ≈ smooth)
- Background: transparent (`setAttribute(Qt.WidgetAttribute.WA_TranslucentBackground)`)

### Window Appearance

```python
window.setWindowFlags(
    Qt.WindowType.FramelessWindowHint |
    Qt.WindowType.WindowStaysOnTopHint
)
window.setAttribute(Qt.WidgetAttribute.WA_TranslucentBackground, False)
window.setFixedSize(360, 240)
```

Center on screen: `window.move(screen.center() - window.rect().center())`

### Button Styling (QSS override on top of qt-material)

```qss
/* Retry button */
QPushButton#retryButton {
    background-color: #26a69a;
    color: #ffffff;
    border-radius: 4px;
    padding: 8px 24px;
    font-size: 10pt;
    font-weight: 600;
}
QPushButton#retryButton:hover {
    background-color: #2bbbad;
}

/* Quit button */
QPushButton#quitButton {
    background-color: transparent;
    color: #ef5350;
    border: 1px solid #ef5350;
    border-radius: 4px;
    padding: 8px 24px;
    font-size: 10pt;
    font-weight: 600;
}
QPushButton#quitButton:hover {
    background-color: rgba(239, 83, 80, 0.1);
}
```
