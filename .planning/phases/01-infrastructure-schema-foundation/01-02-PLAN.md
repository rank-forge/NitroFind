ESHealthWorker contract (D-03, D-04, Pattern 1, Pattern 4):
  class ESHealthWorker(QThread):
    - es_ready = pyqtSignal()             — zero-arg
    - es_failed = pyqtSignal(str)         — reason string
    - __init__(self, es_home: str)
    - run(self)                           — blocking: starts Popen, polls health, emits exactly one signal
    - shutdown_es(self)                   — idempotent; safe from main thread

  Polling parameters (D-04 — non-negotiable):
    - request_timeout = 2 seconds
    - poll interval = 2 seconds
    - total deadline = 60 seconds
    - accept cluster status in ("green", "yellow")
    - URL: http://localhost:9200
    - During the loop, if self.process.poll() returns non-None: emit es_failed("Elasticsearch process exited unexpectedly.") and return
    - On deadline reached: emit es_failed("Elasticsearch did not become healthy within 60 seconds.")

  Subprocess startup (Pattern 2, Pitfall 1):
    - On sys.platform == "win32": pass creationflags=subprocess.CREATE_NEW_PROCESS_GROUP to Popen (REQUIRED so shutdown can use CTRL_BREAK_EVENT)
    - Binary path: os.path.join(es_home, "bin", "elasticsearch")
    - Capture process handle on self.process

Module-level shutdown_es helper (also used directly by tests, per VALIDATION.md test_shutdown_graceful):
  def shutdown_es(process: subprocess.Popen) -> None
    - If process.poll() is not None: return immediately
    - On win32: process.send_signal(signal.CTRL_BREAK_EVENT)
    - Elsewhere: process.terminate()
    - process.wait(timeout=10); on subprocess.TimeoutExpired: process.kill() then process.wait()

  ESHealthWorker.shutdown_es(self) delegates to the module-level shutdown_es(self.process) after a None-guard.

validate_es_home contract (Security Domain V5 / T-es-home mitigation, D-01, D-02):
  def validate_es_home(es_home: str | None) -> str
    - Empty or None: raise ValueError("ES_HOME is not set. Set it to your Elasticsearch 8.18 directory.")
    - Not a directory: raise ValueError("ES_HOME is not a directory: {path}")
    - $ES_HOME/bin/elasticsearch is not a regular file: raise ValueError("Elasticsearch binary not found at: {es_bin}")
    - Otherwise return es_home unchanged

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ES_HOME (shell env) → ESHealthWorker._start_process | Path from untrusted environment crosses into subprocess.Popen — must be validated before exec |
| QThread worker → main GUI thread | Cross-thread signal delivery (pyqtSignal). Qt handles serialization safely; no untrusted data crosses |
| Elasticsearch HTTP API → ES client | All traffic on localhost; security disabled by L-01 (no auth, no TLS) — accepted per product design |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering (path traversal) | nitrofind/es_manager.py::validate_es_home + _start_process | mitigate | `validate_es_home` enforces `os.path.isdir(es_home)` AND `os.path.isfile($ES_HOME/bin/elasticsearch)` before any `subprocess.Popen` call. A malicious `ES_HOME=/tmp/evil` cannot redirect exec to an arbitrary binary because the binary path is hard-coded as `{ES_HOME}/bin/elasticsearch` and validated. |
| T-02-02 | Elevation of Privilege | subprocess.Popen invocation in ESHealthWorker._start_process | mitigate | The Popen command is a list literal `[es_bin]` — no shell=True, no string concatenation. Shell injection via ES_HOME is not possible because the path is passed as a single argv element, not parsed by a shell. |
| T-02-03 | Denial of Service | ESHealthWorker.run() polling loop | mitigate | Hard 60-second deadline via `time.monotonic()` and `time.sleep(2)` between iterations prevents an infinite loop. On process death (`poll() != None`), the worker exits the loop immediately rather than continuing to poll a dead ES. |
