    Class `_SearchWorker(QRunnable)`:
    - `__init__(self, client: Elasticsearch, body: dict, signals: "_SearchSignals") -> None` — stores all three args as instance attributes; calls `super().__init__()`
    - `@pyqtSlot() def run(self) -> None:` — calls `self._client.search(index="car_articles", query=self._body["query"], highlight=self._body.get("highlight"), source=self._body.get("_source"), size=self._body.get("size", 20), from_=self._body.get("from", 0))` using flat kwargs (NO body= parameter); constructs `[ArticleResult.from_es_hit(hit) for hit in resp["hits"]["hits"]]`; emits via `self._signals.results_ready.emit(results)`
    - In except block: `logger.warning("Search failed: %s: %s", type(exc).__name__, exc)` then `self._signals.search_failed.emit(str(exc))`

    Class `SearchEngine`:
    - `__init__(self, client: Elasticsearch) -> None` — stores client; sets `self._pool = QThreadPool.globalInstance()`
    - `search(self, query_text: str, filters: list[dict] | None = None, size: int = 20, callback: Callable[[list], None] | None = None, error_callback: Callable[[str], None] | None = None) -> None`:
      1. `body = build_search_body(query_text, filters=filters, size=size)`
      2. `signals = _SearchSignals()`
      3. `if callback: signals.results_ready.connect(callback)` — BEFORE pool.start
      4. `if error_callback: signals.search_failed.connect(error_callback)` — BEFORE pool.start
      5. `worker = _SearchWorker(self._client, body, signals)`
      6. `self._pool.start(worker)` — LAST, after all signal connections

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| main thread → QRunnable | query_text and filters cross thread boundary inside _SearchWorker constructor |
| _SearchWorker → ES client | Untrusted query_text reaches client.search() inside background thread |
| ES response → main thread | Results cross thread boundary via Qt signal/slot |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-01 | Tampering | _SearchWorker.run() query injection | low | mitigate | query_text is passed via build_search_body into multi_match "query" string value — never interpolated as raw DSL. ES Python client escapes the string safely. |
| T-03-04 | Information Disclosure | _SearchWorker.run() cross-index access | low | mitigate | index="car_articles" is hard-coded as a string literal in _SearchWorker.run(); never derived from user input. |
| T-03-05 | Denial of Service | SearchEngine.search() unbounded size | low | mitigate | build_search_body clamps size to MAX_RESULT_SIZE (100) before it reaches _SearchWorker. Engine layer gets pre-clamped body. |
| T-03-06 | Elevation of Privilege | Qt signal race condition | medium | mitigate | Signals connected BEFORE pool.start(worker) in SearchEngine.search(); enforced by code structure and verified by test order assertions. |
| T-03-SC | Tampering | No new package installs in this plan | n/a | accept | No new packages introduced — all libraries pre-vetted in Phases 1–2. Package legitimacy gate not triggered. |
