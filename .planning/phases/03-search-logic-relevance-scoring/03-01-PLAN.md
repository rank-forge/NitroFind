## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-input → query_text | Untrusted text from search box crosses into ES DSL construction |
| caller → build_search_body size param | Caller-supplied size could exceed safe limits |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-01 | Tampering | build_function_score_query (query_text) | low | mitigate | query_text is always placed inside multi_match "query" string field — never interpolated into raw JSON or DSL path. ES Python client serializes the string safely. No eval(), no string format into DSL keys. |
| T-03-02 | Denial of Service | build_search_body (size param) | low | mitigate | MAX_RESULT_SIZE = 100 constant; build_search_body clamps any caller-supplied size to min(size, MAX_RESULT_SIZE). |
| T-03-03 | Information Disclosure | models.py ArticleResult | low | accept | ArticleResult only exposes fields already in the public ES _source; no internal system paths or secrets. Low-value local desktop app, no PII. |
| T-03-SC | Tampering | No new package installs in this plan | n/a | accept | No new packages introduced — all libraries pre-vetted in Phases 1–2. Package legitimacy gate not triggered. |
</threat_model>
