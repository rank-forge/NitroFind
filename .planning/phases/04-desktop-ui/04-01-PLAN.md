<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ES response → ArticleResult | resp["hits"]["hits"][n]["_source"] crosses from external ES JSON into typed Python dataclass; body field is new attack surface |
| ES response → results_ready signal | resp["took"] integer crosses from ES into a Qt signal payload consumed by the UI |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | ArticleResult.body field populated from ES _source | mitigate | body is consumed only as text content in Qt Rich Text via QTextBrowser (handled in Plan 03); never used as DSL key, never passed to eval/exec, never written to disk |
| T-04-02 | Tampering | resp.get("took", 0) integer extraction | mitigate | took_ms is read from a trusted local ES instance (xpack.security.enabled=false, localhost only per Phase 1 config) and used only to format a status string; falls back to 0 on missing/None |
| T-04-03 | Information Disclosure | _SearchWorker error path | accept | error_callback already mapped to static UI string in Plan 03 (T-04-03 carried forward from main.py); no change in this plan |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan — only existing Phase 1-3 dependencies modified (PyQt6, elasticsearch). RESEARCH.md Package Legitimacy Audit confirms no new packages required for Phase 4. |
</threat_model>
