## Layout Architecture

Primary focal point: SearchLineEdit — receives focus on window open; all other elements are secondary.

### Window Structure

```
QMainWindow (minimum 1100 x 700)
└── central QWidget (QVBoxLayout, spacing=8px, margins=8px all sides)
    ├── Search Row (QHBoxLayout, height=40px fixed)
    │   ├── SearchLineEdit (QLineEdit subclass, expands to fill)
    │   └── [no button — search fires on debounce only]
    ├── Status Label (QLabel, height=20px fixed)
    │   └── "42 results (0.08s)"  [or empty string when idle]
    └── QSplitter (Qt.Orientation.Horizontal, expands to fill)
        ├── FilterSidebar (QWidget, initial width=200px, min=160px, max=240px)
        ├── Result List (QListWidget + ResultDelegate, initial width=320px, min=240px)
        └── Detail Pane (QTextBrowser, initial width=580px, takes remaining space)
```

### Splitter Sizes

```python
splitter.setSizes([200, 320, 580])
```

Rationale: 1100px minimum window — 200 sidebar + 320 list + 580 detail = 1100px.

## Typography

| Role | Size | Weight | Line Height | Applied To |
|------|------|--------|-------------|-----------|
| Result title | 11pt | Bold (700) | 1.2 | Result row title in ResultDelegate HTML |
| Domain label | 9pt | Regular (400) | 1.3 | Source domain line in result row |
| Excerpt | 9pt | Regular (400) | 1.4 | Excerpt line in result row; `<b>` tags from ES are bold at same size |
| Detail heading | 14pt | Bold (700) | 1.2 | Article title in QTextBrowser (`<h2>`) |
| Detail body | 10pt | Regular (400) | 1.5 | Full article text in QTextBrowser |
| Status / UI label | 9pt | Regular (400) | 1.0 | Status label, filter group headers, checkbox labels |
| Search input | 11pt | Regular (400) | 1.0 | QLineEdit text input |

Notes:
- PyQt6 uses `pt` units, not `px`. Qt maps 1pt = 1.333px at 96 DPI.
- Bold weight in HTML fragments must use `font-weight:bold` or `<b>` tag — not numeric weight — because QTextDocument does not support numeric font-weight in inline styles.
- All inline HTML in ResultDelegate uses `style=` attributes; no external CSS.

Source: 04-RESEARCH.md Pattern 3 (_result_to_html example uses `font-size:11pt` and `font-size:9pt`).

## Color

Dark theme is `dark_teal.xml` from qt-material 2.17, applied globally via:
```python
apply_stylesheet(app, theme="dark_teal.xml")
```
This call is already present in `main.py` and executes before any window construction.
Phase 4 must NOT call `apply_stylesheet` again.

### Palette (from dark_teal.xml)

| Role | Hex | Usage |
|------|-----|-------|
| Dominant (60%) | `#1a2327` (approx) | Main window background, QListWidget background, central widget |
| Secondary (30%) | `#2d3b41` (approx) | FilterSidebar background, result row hover state, QSplitter handle |
| Accent (10%) | `#6effe8` (teal) | See reserved list below |
| Destructive | not applicable | Phase 4 has no destructive actions |

Accent `#6effe8` is reserved exclusively for:
1. Selected result row background (Qt selection palette — applied automatically by qt-material)
2. Domain label text color in result rows: `color:#80cbc4` (muted teal variant — confirmed from 04-RESEARCH.md _result_to_html)
3. Search bar focus outline/border (applied automatically by qt-material on QLineEdit focus)

Accent must NOT be used for: general text, unselected result rows, filter labels, status text, article body text.

### Selection Colors (dark_teal.xml confirmed)

| State | Background | Text |
|-------|-----------|------|
| Selected list item | `#6effe8` | `#000000` |
| Unselected list item | transparent / dominant | inherited (light grey) |

Source: 04-RESEARCH.md Pitfall 8 (qt-material GitHub dark_teal.qss confirmed colors).

### HTML Color Overrides in ResultDelegate

The ResultDelegate renders HTML via QTextDocument inside `paint()`. Qt selection palette does NOT propagate into QTextDocument HTML colors. To avoid invisible-text-on-selected-row issues:

- Use `<b>` tags for highlights (bold is visible in both selected and unselected states)
- Do NOT use `background-color` in excerpt HTML — let Qt handle selection background
- Domain color: `color:#80cbc4` is readable on both `#1a2327` background and `#6effe8` selected background

Source: 04-RESEARCH.md Pitfall 8.

## Component Specifications

### 1. SearchLineEdit

**Class:** `SearchLineEdit(QLineEdit)` subclass.

**Overrides:**
- `keyReleaseEvent`: when `event.key() == Qt.Key.Key_Escape` → call `self.clear()`, then `super().keyReleaseEvent(event)`. Use `keyReleaseEvent`, NOT `keyPressEvent` (Escape only fires on release — hardware-dependent cross-platform quirk).

**Properties:**
- `placeholderText`: `"Search cars, manufacturers, models…"` (see Copywriting Contract)
- `clearButtonEnabled`: `True` — shows Qt native clear (×) button
- Height: 40px (set via `setFixedHeight(40)`)
- Font: 11pt regular

**Debounce wiring (in MainWindow.__init__):**
```python
self._debounce_timer = QTimer(self)
self._debounce_timer.setSingleShot(True)
self._debounce_timer.setInterval(300)  # SRCH-01: 300ms
self._debounce_timer.timeout.connect(self._execute_search)
self._search_bar.textChanged.connect(self._debounce_timer.start)
```
Calling `QTimer.start()` when already running restarts the countdown — this is correct debounce behavior.

Source: 04-RESEARCH.md Pattern 1.

### 3. ResultDelegate

**Class:** `ResultDelegate(QStyledItemDelegate)` in `nitrofind/ui/result_delegate.py`.

**Row padding:** `QMargins(8, 8, 8, 8)` — 8px sides, 8px top/bottom. `_ROW_PADDING = QMargins(8, 8, 8, 8)`.

**HTML template per result row:**
```python
def _result_to_html(result: ArticleResult) -> str:
    title = result.highlight_title[0] if result.highlight_title else result.title
    excerpt = result.highlight_body[0] if result.highlight_body else result.excerpt
    return (
        f"<b style='font-size:11pt'>{title}</b>"
        f"<br><span style='color:#80cbc4;font-size:9pt'>{result.source_domain}</span>"
        f"<br><span style='font-size:9pt'>{excerpt}</span>"
    )
```

**`_make_doc(html, width)` helper:** Used identically by both `paint()` and `sizeHint()` to prevent height mismatch. Must set `textWidth`, `documentMargin=0`, and `WrapAtWordBoundaryOrAnywhere` in both methods.

**Data roles in QListWidgetItem:**
- `Qt.ItemDataRole.UserRole`: HTML string (for rendering)
- `Qt.ItemDataRole.UserRole + 1`: `ArticleResult` object (for detail pane on selection)

Source: 04-RESEARCH.md Pattern 3, Pitfall 3.

### 4. Result List (QListWidget)

**Class:** `QListWidget` instance in MainWindow, with `ResultDelegate` set via `setItemDelegate()`.

**Properties:**
- `uniformItemSizes`: `False` — rows have variable height (multi-line HTML)
- `selectionMode`: `QAbstractItemView.SelectionMode.SingleSelection`
- `horizontalScrollBarPolicy`: `Qt.ScrollBarPolicy.ScrollBarAlwaysOff`

**Signal connections:**
```python
self._result_list.currentRowChanged.connect(self._on_result_hovered)
self._result_list.itemActivated.connect(self._on_result_activated)
```
`itemActivated` fires on both double-click AND Return/Enter key press — covers UIPL-04 Enter-to-open requirement.

Arrow key navigation is built into `QListWidget` with no additional code. Arrow keys change `currentRow`, which fires `currentRowChanged`, which updates the detail pane.

**Populate sequence:**
```python
self._result_list.clear()
for r in results:
    item = QListWidgetItem()
    item.setData(Qt.ItemDataRole.UserRole, _result_to_html(r))
    item.setData(Qt.ItemDataRole.UserRole + 1, r)
    self._result_list.addItem(item)
```

### 5. FilterSidebar

**Class:** `FilterSidebar(QWidget)` in `nitrofind/ui/filter_sidebar.py`.

**Layout:** `QVBoxLayout` with `QLabel` group headers and `QCheckBox` items per group.

**Filter groups and hardcoded values (Phase 4 — ES aggregations deferred to v2):**

| Group label | Checkboxes | ES field | `build_filter_clauses` arg |
|-------------|-----------|----------|---------------------------|
| "Manufacturer" | Ferrari, Porsche, BMW, Ford, Chevrolet, Toyota, Honda, Lamborghini, McLaren, Aston Martin | `manufacturer` | `manufacturer=` |
| "Era" | 1950s, 1960s, 1970s, 1980s, 1990s, 2000s, 2010s, 2020s | `era_bucket` | `era_bucket=` |
| "Body Style" | Coupe, Sedan, Convertible, SUV, Hatchback, Wagon, Pickup, Van | `body_style` | `body_style=` |

**Selection model:** Single-select per group (radio-button behavior via mutual exclusion in Python state — use `QButtonGroup` with `setExclusive(True)` for each group, or track checked state manually and uncheck siblings on `stateChanged`). Clicking a checked box again unchecks it (deselect → no filter on that dimension).

**Filter persistence (SRCH-04):** CheckBox state is NOT cleared when the user types a new query. The `_execute_search()` method always calls `self._filter_sidebar.collect_filters()` at fire time — whatever is checked at that moment is applied to the new query.

**Signal wiring:** Each `QCheckBox.stateChanged` connects to `self._debounce_timer.start` in MainWindow (same as search bar) so filters trigger a new search immediately with 300ms debounce.

**`collect_filters()` return contract:**
```python
def collect_filters(self) -> list[dict]:
    # Returns result of build_filter_clauses(manufacturer, era_bucket, body_style)
    # where each arg is the checked value (str) or None if none selected
```

### 7. Stale Result Guard

**Location:** MainWindow instance variable + `_on_results()` method.

```python
self._current_seq: int = 0

def _execute_search(self) -> None:
    self._current_seq += 1
    seq = self._current_seq
    self._engine.search(
        self._search_bar.text().strip(),
        filters=self._filter_sidebar.collect_filters(),
        callback=lambda results, took, s=seq: self._on_results(results, took, s),
        error_callback=self._on_search_error,
    )

def _on_results(self, results: list, took_ms: int, seq: int) -> None:
    if seq != self._current_seq:
        return  # stale — discard
    count = len(results)
    self._status_label.setText(f"{count} results ({took_ms / 1000:.2f}s)")
    self._populate_results(results)
```

Source: 04-RESEARCH.md Pattern 2, Pitfall 1.

### 8. Results-Ready Signal Extension (Wave 0 required change to Phase 3)

The current `_SearchSignals.results_ready` emits `pyqtSignal(list)` (list only). UIPL-02 requires displaying query time. Phase 4 Wave 0 must change the signal signature:

```python
# In nitrofind/search/engine.py:
results_ready = pyqtSignal(list, int)  # list[ArticleResult], took_ms (integer ms)

# In _SearchWorker.run():
took_ms = resp.get("took", 0)
self._signals.results_ready.emit(results, took_ms)
```

All existing consumers of `results_ready` (test_engine.py) must be updated to accept the new `(list, int)` signature. This is a required Wave 0 task — plan must include it explicitly.

Source: 04-RESEARCH.md Open Question 3, Code Examples (Reading ES `took`).

## Interaction Contracts

### Debounce (SRCH-01)

| Event | Action |
|-------|--------|
| User types a character | `QTimer.start()` restarts the 300ms countdown |
| 300ms pass with no keystrokes | `QTimer.timeout` fires → `_execute_search()` |
| Empty query (len == 0 after strip) | `_execute_search()` must call `self._result_list.clear()` and set status to `""` without dispatching to ES |
| Filter checkbox changes | Same debounce path — checkbox `stateChanged` → `_debounce_timer.start` |

### Keyboard Navigation (UIPL-04)

| Key | Widget | Action |
|-----|--------|--------|
| Up / Down arrows | QListWidget | Move `currentRow` (Qt built-in) → `currentRowChanged` → update detail pane |
| Enter / Return | QListWidget | `itemActivated` fires → `_on_result_activated()` |
| Escape | SearchLineEdit | `keyReleaseEvent` → `self.clear()` → `textChanged` fires → debounce fires → `_execute_search()` clears results |
| Tab | SearchLineEdit → QListWidget | Qt default focus traversal (no override needed) |

### Result Selection Flow

```
currentRowChanged(row)
  → item = self._result_list.item(row)
  → result: ArticleResult = item.data(Qt.ItemDataRole.UserRole + 1)
  → self._detail_pane.setHtml(...)
```
This handles both arrow-key hover and click selection. Detail pane updates on hover, not only on activation.

### Filter Persistence (SRCH-04)

Filter checkboxes are owned by `FilterSidebar` and are NOT reset on search. `_execute_search()` reads `collect_filters()` on every call — active filters apply to every new query automatically.

## State Mapping

| State | Trigger | SearchBar | StatusLabel | ResultList | DetailPane |
|-------|---------|-----------|-------------|------------|------------|
| Idle (no search yet) | App opens | Placeholder text visible | `""` (empty) | Empty | "Select a result to read the article." |
| Typing / debounce active | User keystroke | Text typed | `"Searching…"` | Previous results or empty | Unchanged |
| Loading (ES in flight) | Debounce fires | Text typed | `"Searching…"` | Previous results or empty | Unchanged |
| Results ready | `_on_results()` | Text typed | `"N results (0.Xs)"` | Populated | Unchanged until selection |
| No results | `_on_results([])` | Text typed | `"No results"` | Empty + empty state label | "No articles match your search. Try different keywords." |
| Error | `_on_search_error()` | Text typed | `"Search failed. Check Elasticsearch connection."` | Unchanged (previous) | Unchanged |
| Result selected | `currentRowChanged` | Unchanged | Unchanged | Row highlighted | Article detail HTML |
| Escape pressed | `keyReleaseEvent` | Cleared | `""` | Cleared | "Select a result to read the article." |

### Empty State (result list)

When `_on_results([])` is called, in addition to setting the status label, display a centered placeholder message inside the result list area. Use a `QLabel` overlaid on the `QListWidget` (show/hide based on list count), or set a disabled `QListWidgetItem` with the copy.

Copy: `"No articles match your search.\nTry different keywords or adjust filters."`
