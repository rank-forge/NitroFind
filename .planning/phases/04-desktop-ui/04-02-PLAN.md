<objective>
Build the two reusable, independent UI components consumed by MainWindow in Plan 03: the QStyledItemDelegate that renders each result row as multi-line HTML (with <b> highlight tags), and the QWidget filter sidebar with three single-select QButtonGroup-backed groups (Manufacturer, Era, Body Style). Both components are self-contained — no SearchEngine dependency, no MainWindow dependency — so they can be unit-tested in isolation with pytest-qt qtbot and a MagicMock ArticleResult, then consumed by Plan 03.

Purpose: SRCH-02 (result rows show title/domain/highlighted excerpt) and UIPL-01 (query terms highlighted) live entirely in the delegate. SRCH-04 (sidebar filters narrow results without clearing query) lives entirely in the sidebar's collect_filters contract. Building them first means Plan 03 only has to wire — not author — these surfaces.

Output: nitrofind/ui/result_delegate.py with ResultDelegate + _result_to_html, nitrofind/ui/filter_sidebar.py with FilterSidebar, and pytest-qt unit tests covering the SRCH-02, SRCH-04, UIPL-01 behaviors enumerated in 04-VALIDATION.md.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ArticleResult fields → HTML rendered by QTextDocument | result.title, result.source_domain, result.excerpt, result.highlight_title[0], result.highlight_body[0] are concatenated into HTML and rendered by Qt Rich Text in the result list delegate |
| Filter checkbox state → ES query (via build_filter_clauses) | Checked checkbox keys (str) cross from UI state into ES term query values |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Tampering (XSS) | _result_to_html HTML rendering | mitigate | Output is rendered exclusively by Qt Rich Text via QTextDocument inside QStyledItemDelegate — no QtWebEngine, no <script> execution; Qt Rich Text supports a controlled HTML subset (no JS, no event handlers). RESEARCH.md Security Domain confirms this is the standard mitigation. |
| T-04-05 | Tampering | Filter checkbox values passed to build_filter_clauses | mitigate | Filter values are hardcoded module-level tuples (MANUFACTURERS, ERAS, BODY_STYLES) in filter_sidebar.py — never user input. build_filter_clauses (Phase 3) places them in ES term query values, never as DSL keys (T-03-01 enforced). |
| T-04-06 | Denial of Service | ResultDelegate paint() per-row QTextDocument rebuild | accept | RESEARCH.md notes per-paint rebuilds acceptable for ~20 result page size. If result counts exceed 50, caching becomes necessary — out of scope for v1. |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan — uses existing Phase 1-3 dependencies (PyQt6 6.11.0, qt-material 2.17). RESEARCH.md Package Legitimacy Audit confirms no new installs required for Phase 4. |
</threat_model>
