## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem -> config loader | `config/scraper.yaml` is committed source-controlled content edited by the developer; no untrusted producer in scope for this plan. Pyyaml is added now; consumers must use `yaml.safe_load()` (enforced in Plan 02 + Plan 05). |
| pip -> venv | New packages (mediawikiapi, beautifulsoup4, lxml, pyyaml) are pinned with hashes via pip-compile, preventing supply-chain swap. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | `config/scraper.yaml` -> future YAML loader | mitigate | Yaml.safe_load() is mandated by Plan 02 and Plan 05; this plan ships pyyaml>=6.0 so safe_load is available. No `yaml.load()` may appear in any scraper module — enforced by acceptance criteria in later plans. |
| T-02-02 | Tampering | requirements.txt supply chain | mitigate | `pip-compile --generate-hashes` produces sha256 hashes for every package; `pip install -r requirements.txt` refuses mismatching wheels. |
| T-02-03 | Information Disclosure | `data/` runtime SQLite DB | mitigate | `.gitignore` entry `data/` prevents accidental commit of `scraper_state.db` which contains visited Wikipedia page IDs (not sensitive but bulky and noisy). |
| T-02-04 | Tampering | Future User-Agent header in scraper.yaml | accept | Honest `User-Agent` value `NitroFind/1.0 (...)` is set in config — no browser impersonation. If Hagerty blocks honest UA, Plan 04 logs warning and falls back; we do not bypass bot protection (RESEARCH.md security domain). |
</threat_model>
