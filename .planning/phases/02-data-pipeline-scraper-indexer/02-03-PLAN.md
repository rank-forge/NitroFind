<objective>
Build the Wikipedia data source: a `WikipediaScraper` class that walks category trees from `config/scraper.yaml` root categories, paginates large categories via raw MediaWiki API with cmcontinue, fetches pages by pageid (not title), filters out pages without infoboxes, and yields document dicts conforming to `CAR_ARTICLES_MAPPING`. The scraper integrates with the Plan 02 state manager (skip already-visited pageids per D-06) but does NOT integrate with the indexer — that wiring happens in Plan 05.

Purpose: SCRP-01 is the highest-confidence requirement (Wikipedia stack is fully verified per RESEARCH.md), can be built in parallel with the blog scraper (Plan 04), and is the source of the 1,000+ infobox articles that satisfy the phase success criterion. By yielding documents (generator pattern) rather than directly indexing, the scraper can be unit-tested in isolation and composed with `build_action` + `BulkIndexer` in the CLI plan.

Output: A single new module (`nitrofind/scraper/wikipedia.py`) with one class and unit tests covering category walk, infobox filter, and document dict construction. Live API calls only in integration tests (marked `@pytest.mark.integration`).
</objective>
