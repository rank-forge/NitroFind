# Milestone v1.0: NitroFind MVP

**Status:** ✅ SHIPPED 2026-05-29
**Phases:** 1–5
**Total Plans:** 18

## Overview

NitroFind v1.0 is the complete first release of an offline desktop application for automotive enthusiasts. Starting from zero, this milestone delivered a full working stack: a reproducible Python environment with a local Elasticsearch 8.18 node, a one-shot CLI scraper that populates the index from Wikipedia and automotive blogs, a `function_score` search engine with composite quality-based ranking, a PyQt6 dark-teal native UI with instant search and filter sidebar, and a PyInstaller distribution bundle that ships alongside a pre-extracted Elasticsearch directory.

## Milestone Summary

**Decimal Phases:** None

**Key Decisions:**
- Horizontal-layer phase structure — each phase is a complete technical subsystem (Infrastructure → Pipeline → Search → UI → Packaging)
- ES 8.18 with xpack.security.enabled: false, network.host: 127.0.0.1, JVM heap pinned at 512 MB
- car_articles index with dynamic: false and flattened infobox type to prevent mapping explosion
- State dict pattern `state = {"worker": None}` in main.py so closures can replace ESHealthWorker without nonlocal
- ES cold-start deadline extended from 60s (plan spec) to 180s — observed ~120s actual cold-start on target machine
- function_score: Gaussian decay (730d scale) + missing-field fallback (0.3 weight) + log1p(word_count) + has_infobox boost; score_mode: sum, boost_mode: multiply
- PyInstaller onedir mode (not --onefile): avoids AV false-positives and per-launch extraction delay
- collect_all('qt_material') required — PyInstaller has no built-in hook for qt_material
- upx=False in both EXE and COLLECT: UPX corrupts Qt DLLs on Windows
- ES_BUNDLE env var for assembly: keeps ES path out of spec for repeatable builds

**Issues Resolved:**
- Phase 1 Plan 04: Retry button required state dict to replace active ESHealthWorker from within closure — nonlocal not sufficient; state dict pattern used
- Phase 2: Blog parser CSS selectors implemented despite MEDIUM confidence on selector stability; HTTP 403 fallback chain added for resilience
- Phase 5: DEVNULL-hardened Popen required because console=False suppresses stdout/stderr in frozen mode — orphaned handles would cause subprocess to hang

**Issues Deferred:**
- Phase 3 live ranking quality validation (Ferrari 308 top-3, recency/infobox separation) — deferred to v1.1; requires live indexed data for empirical tuning
- Phase 5 clean-machine Windows smoke test — deferred; requires machine without Python/Java installed
- function_score weights need empirical tuning against real indexed data — weights from literature are starting points

**Technical Debt Incurred:**
- REQUIREMENTS.md checkboxes were never updated during phase execution — corrected in this archive
- Windows NSIS installer (UI-V2-01) not built — onedir zip used for v1.0; proper installer deferred to v2
