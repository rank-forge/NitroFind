## Overview

Replaced the PyQt6 desktop UI with a Flask web server accessible at `http://localhost:5000`. A single `python main.py` now starts Elasticsearch and the web server together. The v1.0 scraper and search engine were preserved without rewrite; the entire UI tier was replaced with a browser-based single-page app. PyQt6 and qt-material fully removed from the dependency tree.

## Milestone Summary

**Key Decisions:**

- PyQt6/qt-material removed; Flask dev server is the v1.1 distribution model — no PyInstaller bundle needed
- Linux/WSL-only for v1.1 — all `sys.platform == 'win32'` branches removed from `es_manager.py`
- Module-level state dict pattern extended to `es_client` (GIL-safe single writer)
- `data-state` CSS attribute selectors for SPA view switching — no JS `show/hide` on DOM elements
- System-font stack over self-hosted web fonts — offline-safe, zero setup (D-08)
- `took_ms` as per-item field in JSON array rather than envelope — satisfies API-01 wording literally

**Issues Resolved:**

- PyQt6 dependency eliminated — app no longer requires Qt install
- Flask template resolution pitfall: `_pkg_dir` absolute path used for `template_folder`/`static_folder` because `Flask(__name__)` inside `nitrofind/server.py` resolves root_path to `nitrofind/` not project root

**Issues Deferred:**

- Empirical tuning of `function_score` weights against real indexed data (carried from v1.0)
- Windows clean-machine smoke test (carried from v1.0 — Linux/WSL only for v1.1)
- `Manufacturer` filter uses free-text only (no aggregation endpoint for dropdown population)

**Technical Debt Incurred:**

- `took_ms` per-item rather than envelope — Phase 8 migrated to `envelope.took_ms` at display layer; backend still per-item
- `PYQT6_AVAILABLE` skip guard left in `tests/test_search/test_engine.py` — can be removed once PyQt6 fully confirmed absent
