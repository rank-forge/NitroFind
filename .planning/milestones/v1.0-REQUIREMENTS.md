# Requirements: NitroFind v1.0

**Defined:** 2026-05-12
**Core Value:** Instant, noise-free access to deep automotive knowledge — the entire database on your machine, searchable in milliseconds.

## v1 Requirements

### Infrastructure

- [x] **INFRA-01**: Developer can run the app from a Python venv with a pinned lockfile (reproducible environment across machines) — *Implemented Phase 1, pip-compile + SHA256 hashes*
- [x] **INFRA-02**: App starts a local Elasticsearch 8.18 node as a subprocess on launch (localhost:9200, TLS disabled, security disabled) — *Implemented Phase 1, ESHealthWorker + subprocess.Popen*
- [x] **INFRA-03**: App terminates the Elasticsearch process cleanly when the user quits — *Implemented Phase 1, shutdown_es() cross-platform*
- [x] **INFRA-04**: App shows a loading state while Elasticsearch warms up and becomes healthy — *Implemented Phase 1, LoadingWindow + SpinnerWidget state machine*

### Data Schema

- [x] **SCHEMA-01**: Each indexed document contains core identity fields: title, url, source_domain, article_id, scraped_at — *Implemented Phase 1, car_articles mapping*
- [x] **SCHEMA-02**: Each indexed document contains relevance scoring fields: published_at, word_count, has_infobox, image_count — *Implemented Phase 1, car_articles mapping*
- [x] **SCHEMA-03**: Each indexed document contains full plain-text body (for full-text search) and a 300-character excerpt (for result list display) — *Implemented Phase 1 + Phase 2*
- [x] **SCHEMA-04**: Each indexed document contains automotive facet fields: manufacturer, production_start, production_end, body_style, era_bucket, country_of_origin — *Implemented Phase 1 + Phase 2*

### Scraper

- [x] **SCRP-01**: Scraper fetches car articles from Wikipedia using the MediaWiki API (structured JSON, not HTML parsing) — *Implemented Phase 2, WikipediaScraper + mediawikiapi*
- [x] **SCRP-02**: Scraper fetches articles from at least one automotive blog using BeautifulSoup4 — *Implemented Phase 2, BlogScraper (Hagerty)*
- [x] **SCRP-03**: Scraper uses MediaWiki page ID as the Elasticsearch document `_id` to prevent duplicate articles — *Implemented Phase 2*
- [x] **SCRP-04**: Scraper stops indexing and logs a warning when the index approaches 1.8 GB — *Implemented Phase 2, BulkIndexer size guard*

### Relevance Scoring

- [x] **RLVN-01**: Search results ranked using Elasticsearch function_score with Gaussian recency decay (~2-year half-life) — *Implemented Phase 3, 730d scale Gaussian*
- [x] **RLVN-02**: function_score includes article length signal using logarithmic modifier on word_count — *Implemented Phase 3, log1p modifier*
- [x] **RLVN-03**: function_score includes boolean boost for articles with structured infobox (has_infobox = true) — *Implemented Phase 3*
- [x] **RLVN-04**: Relevance signals combined additively (score_mode: sum); text match acts as multiplier (boost_mode: multiply) — *Implemented Phase 3*

### Search Interface

- [x] **SRCH-01**: Search results update as the user types, with 300ms debounce — *Implemented Phase 4, single-shot QTimer*
- [x] **SRCH-02**: Result list displays title, source domain, and highlighted excerpt — *Implemented Phase 4, ResultDelegate + _result_to_html*
- [x] **SRCH-03**: Selecting a result displays full article text inline in a detail pane (no browser) — *Implemented Phase 4, QTextBrowser + setOpenLinks(False)*
- [x] **SRCH-04**: Filter sidebar lets user narrow results by manufacturer, era_bucket, and body style without clearing query — *Implemented Phase 4, FilterSidebar + collect_filters()*

### UI Quality

- [x] **UIPL-01**: Query terms highlighted in result excerpts — *Implemented Phase 4, ES highlight tags rendered via QTextDocument*
- [x] **UIPL-02**: Result count and query time displayed below search box — *Implemented Phase 4, "N results (T:.2fs)" format*
- [x] **UIPL-03**: UI ships with dark theme as default — *Implemented Phase 4, apply_stylesheet(app, "dark_teal.xml")*
- [x] **UIPL-04**: User can navigate with arrow keys, open with Enter, clear with Escape — *Implemented Phase 4, keyReleaseEvent*

### Packaging

- [x] **PKG-01**: App distributed as PyInstaller bundle alongside pre-extracted Elasticsearch directory (no Python or Java install required) — *Implemented Phase 5, nitrofind.spec + scripts/build_dist.py*

## v2 Requirements

### Filters (Extended)

- **FILT-01**: Filter by engine type (inline-4, V8, flat-6, rotary, etc.)
- **FILT-02**: Filter by country of manufacture
- **FILT-03**: Filter by drivetrain (RWD, FWD, AWD, 4WD)

### Relevance (Extended)

- **RLVN-V2-01**: Domain authority tier weight (manual 1–5 per source domain) as an additional function_score signal
- **RLVN-V2-02**: Inbound link count signal (sqrt modifier) for cross-referenced authority

### Data

- **DATA-01**: Database update workflow — re-run scraper and incrementally re-index without full wipe
- **DATA-02**: Source whitelist/blacklist configuration file for controlling which domains are scraped

### UI

- **UI-V2-01**: Full installer with desktop shortcut (NSIS on Windows, .dmg on macOS)
- **UI-V2-02**: Light/dark theme toggle
- **UI-V2-03**: Search history (last N queries, persisted locally)
- **UI-V2-04**: Bookmarking — save articles to a local favorites list

## Out of Scope

| Feature | Reason |
|---------|--------|
| Motorcycles, trucks, other non-car vehicles | Keeps database under 2 GB; aligns with enthusiast focus |
| AI/ML-based relevance (embeddings, semantic search) | Deliberate design choice — deterministic math only |
| Online/cloud sync or API calls at search time | Offline-first is the core product promise |
| User accounts or authentication | Single-user local tool |
| Real-time database auto-update | Scraper is a one-shot tool; manual re-run is sufficient |
| Web scraping non-automotive content | Out of domain scope |
